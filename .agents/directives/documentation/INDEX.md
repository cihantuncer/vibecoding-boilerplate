# Documentation Directives Index

Use for README, docs, examples, comments, plans, roadmap, and explanations.

- Identify audience, durability, and source of truth before writing.
- Keep docs accurate, navigable, and close to the thing they describe.
- Prefer concise examples over broad prose.
- Do not duplicate volatile details unless maintenance cost is acceptable.

## Route Map

| Signal | Directive | Open when |
|---|---|---|
| small README/comment/example edit | none | this index is enough |
| architecture, product decision, roadmap, operational process | `./documentation-durable-context.md` | docs become durable project context |
| public or multi-file documentation risk | `./documentation.md` | risk is not covered by this index |

Open at most one documentation directive unless the task clearly needs more.
