# Durable Documentation Directive

Define audience and source of truth before writing. Separate decisions, assumptions, options, and next steps. Link to volatile details instead of copying them when they may drift.

Keep durable docs short and navigable. Use examples only when they reduce ambiguity.

Validate names, paths, commands, and claims against current files. Mark uncertainty explicitly instead of presenting guesses as facts.
